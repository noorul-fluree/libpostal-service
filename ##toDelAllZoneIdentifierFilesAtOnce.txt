Removing Windows :Zone.Identifier Files in WSL

When working on WSL with files copied from Windows, you may notice extra files like:

README.md:Zone.Identifier
setup-phase1.sh:Zone.Identifier


These are Windows Alternate Data Streams (ADS)—they are harmless but clutter your project and sometimes cause duplication or git issues.

This guide shows how to delete all :Zone.Identifier files recursively.

Step 1: Navigate to your project folder

Use cd to go to the folder where you want to clean up:

cd ~/FAISS-Actual-26June25/2-20March-address-service-optimized-WorkingGudJsonBodyBkp/



Step 2: Check which files will be deleted

Before deleting anything, it’s safe to list all :Zone.Identifier files:

find . -type f -name "*:Zone.Identifier"


Example output:
./README.md:Zone.Identifier
./src/controllers/ParseController.cc:Zone.Identifier
./build/Makefile:Zone.Identifier
./logs/drogon.260319-095315.000000.log:Zone.Identifier


✅ Only files ending with :Zone.Identifier are listed.



Step 3: Delete all :Zone.Identifier files recursively

Once you’ve verified the list, delete them using:

find . -type f -name "*:Zone.Identifier" -delete


Explanation:

* find . → start search from current directory

* -type f → match only files (not directories)

* -name "*:Zone.Identifier" → match only files ending with :Zone.Identifier

* delete → remove the matched files

This works recursively through all subfolders (src, logs, build, etc.) and does not touch your actual project files.



Step 4: Verify deletion

	To make sure all :Zone.Identifier files are gone:
	find . -type f -name "*:Zone.Identifier"

	If nothing is returned → success ✅

	Your folder is now clean.

Step 5: Optional – Example

	Suppose your folder looks like this before deletion:

README.md
README.md:Zone.Identifier
src/main.cc
src/main.cc:Zone.Identifier
logs/error.log
logs/error.log:Zone.Identifier

After running the delete command:

README.md
src/main.cc
logs/error.log

All :Zone.Identifier files are gone, original files remain intact.
